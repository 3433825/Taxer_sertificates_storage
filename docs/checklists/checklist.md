# 1. Стартова сторінка

## UI / Layout
- Відображення кнопки додавання сертифікату
- Назва кнопки відповідає макету
- Наявність тексту-заглушки при порожньому списку
- Коректне розташування елементів
- Відсутність візуальних дефектів (зсуви, накладання)

## Стан списку
- Відображення 0 сертифікатів (empty state)
- Відображення 1 сертифікату
- Відображення кількох сертифікатів
- Відображення активного (selected) сертифікату
- Коректне оновлення списку після додавання

## Взаємодія
- Клік по сертифікату активує його
- Активний елемент має візуальний індикатор
- Поведінка при повторному кліку

# 4. Drag & Drop

## Базова функціональність
- Завантаження валідного .cer файлу
- Відображення індикатора обробки
- Додавання сертифікату у список

## Негативні сценарії
- Невалідний формат (.txt, .jpg)
- Пошкоджений файл
- Порожній файл
- Файл без сертифікатних даних

## Edge cases
- Drag кількох файлів одночасно
- Повторний drag того самого файлу
- Drag під час обробки іншого файлу
- Drop поза зоною
- Дуже великий файл

# 6. Валідація сертифікату (Data validation)

## Структура
- Наявність обовʼязкових полів (CN, Issuer, Dates)
- Обробка відсутніх полів

## Дані
- Дуже довгий CN (>50 символів)
- Дуже довгий Issuer
- Спецсимволи / кирилиця / unicode
- HTML / XSS payload у полях

## Дати
- Valid From < Valid To
- Прострочений сертифікат
- Сертифікат з майбутньою датою

# 9. LocalStorage

## Базова поведінка
- Збереження після reload
- Збереження після закриття вкладки
- Відновлення списку

## Edge cases
- Очищений LocalStorage
- Пошкоджені дані (manual edit)
- Невалідний JSON
- Частково збережені дані

## Обмеження
- Поведінка при великому обʼємі даних (~5MB)
- Обробка помилки запису

# 10. Граничні умови

- 0 сертифікатів

- Дуже довгі текстові значення
- Мінімальні/порожні значення
